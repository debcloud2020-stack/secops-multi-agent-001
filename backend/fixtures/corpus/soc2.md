# SOC 2 Trust Services Criteria — common criteria summaries (paraphrased)

## CC6 — Logical and physical access controls
- CC6.1 Restrict logical access; enforce least privilege and strong authentication.
- CC6.2 Register and authorize new access before granting credentials; revoke promptly.
- CC6.6 Protect against threats from outside the system boundary (e.g., internet-facing
  services), including segmentation of high-risk assets.

## CC7 — System operations
- CC7.1 Detect and monitor for vulnerabilities and anomalies; maintain detection tooling.
- CC7.2 Monitor system components for anomalous activity (e.g., privilege escalation,
  unusual sign-ins).
- CC7.3/CC7.4 Evaluate security events and respond per the incident response plan,
  including containment and remediation within defined timeframes.

## CC8 — Change management
Authorize, design, develop, test, and approve changes; apply security patches under a
defined SLA.
